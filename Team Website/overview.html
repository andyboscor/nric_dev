Overview

Background, Problem, Requirements and Use Cases

Background and Problem

The International Resource for Infection Control (iNRIC) represents a very important part of healthcare (founded with WHO, ECDCC, HPA) that is used across the world by various medical staff due to it’s extensive database of articles, news and links gathered from WHO, ECDC and many other sources. The main website (nric.org.uk) is currently used and continuously updated with information.

The website is responsive and can work on mobile platforms, however it does not allow any sort of personalised customisation for each individual user. Furthermore, underdeveloped countries are unlikely to be able to access the desktop version, so they would need an easier to use app than the mobile version of the website.

Initial Requirements

Our team decided together with our client to focus on the design and development of the application first and integrate user profilling on the website later on.

Must
  • application must be an easy-to-use version of the website
  • must use Android and iOS as main platforms
  • have User Profile on main desktop website with registration and article preferences
  • have User Profile on application with User Preferences panel
  • have Latest Articles and Headlines tailored to user preferences (By Country and chosen Article Preferences)
  • must use Drupal website database
Should
  • have push notifications on both platforms
  • maintain main website look
  • knowledge week transfer of data from old nric website to new one
Could
  • transfer newsletter feature from old website to new one MailChimp
    Won't
  • none

Functional Requirements

ID Description Category Platform Priority
F1 The system shall allow the user to register. Account App/Website Must
F2 The system shall allow the user to log in and log out. Account App/Website Must
F3 The system shall save the user’s information in the Drupal database (shared between website and app). Account App/Website Must
F4 The system shall allow the user to reset their password via email. Account App/Website Must
F5 The system shall allow the user to save and change Article Preferences to their account. Account App/Website Must
F6 The system shall allow the user to change their account details. Account App/Website Must
F7 The system shall allow the user to add a profile picture to their account. Account App/Website Should
F8 The system shall present the user with a list of articles based on the user’s Preferences. Articles App Must
F9 The system shall present the user with a list of articles based on the user’s Preferences. Articles Website Should
F10 The system shall allow the user to contribute Articles. Articles App Must
F11 The system shall allow the user to search for articles based on Account Preferences. Search App Must
F12 The system shall allow the user to search for articles based on Filters (ignoring account preferences). Search App Must
F13 The system shall allow the user to search for News. Search App Must
F14 The system shall redirect old Knowledge Week data from old website to the existing one. Knowledge Week Website Should
F15 The system shall have a newsletter feature with MailChimp. Newsletter Website Could
F16 The system shall notify the user of new articles. Notifications App Must

Non-Functional Requirements

ID Description Platform Priority
NF1 The system shall maintain the aspect of the website. App Must
NF2 The system shall be available on iOS and Android. App Must
NF3 The system shall maintain the aspect of the website. App Must
NF4 The system shall be coded in HTML5, CSS3, Javascript and will be ported on mobile platforms with Phonegap. App Must
NF5 The system shall be available 24 hours a day, 7 days a week. App Must
NF6 The system database shall be shared between the website and the application. App Must
