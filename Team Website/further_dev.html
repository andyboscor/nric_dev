SDK/Collaboration Opportunities

The app is built in native HTML5, CSS3, Javascript (jQuery) code and is easy to modify if needed. Furthermore anything added in the main iNRIC website can also be added to the app as a AJAX requests to the endpoint. There are HTML and CSS classes already built to support integration of more information in the app itself. Furthermore, the app can also become very secure once the previous developer fixes the theme that gets broken upon turning the protocol into HTTPS.

Even though they are not yet working (due to Phonegap Build plugin issue), the notifications could be easily used to report outbreaks to various medical staff around the world.

There is also the possibility to use data within the app (number of clicks on certain articles, number of searches on certain topics) to get relevant information for Healthcare. Furthermore, the app could possibly integrate even more Health Systems (OpenEHR, ECDC, etc).

As iNRIC is an international resource with over a quarter million users from 159 different countries, it is already quite a popular product. The app is an extension of this website, meant to accommodate people’s shift to mobile as well as offer a low data option for less developed areas (AJAX requests use very little data compared to actual website). For this reason, the app has been presented to very important people from the Digital Health sector at the Festival For Digital Health held at Digital Catapult.

User Manual

  1. Login

    • This is the first page presented from the app. Simply add your username and password to log in.

  2. Forgot Password

    • In case you forgot the password to your account, all you have to do is press the “Forgot your password?” button and input your email. You will receive an email to change your password.

  3. Registration

    • In case you don’t already have an account from iNRIC, you can create a new one by pressing the register “Button”. You will receive an email to set up a password.

  4. Home Page

    • You will be presented with a Home Page that contains Headlines, Latest News and Recently Added updated directly from iNRIC. Click on any link to open the in-app browser.

  5. Search Page

    • Choose any search key to query the iNRIC resource database. Furthermore you can also choose a category or apply Filters to your search.
    • You can use Quick Search with your previously saved Preferences. (We’ll tell you how to save them later on)
    • You can also contribute your own resources to the database by using the Contribute part of the page.

  6. In Focus

    • This page contains the resources regarding Infection Weeks and Training.

  7. Links

    • This page provides useful links and conferences. ( Also updated from iNRIC)

  8. Profile

    • This page contains your Preferences which you can always update.
    • You can also change your email, country or request a new password.
    • You can also logout.

System Manual

All *.html pages use index.css (contains all CSS classes).

  1. Login Page

    • The login page is in path “www/login.html”. It contains both the HTML5 and Javascript code used.
    • The login function contains a request to the Drupal RESTful API. (Implementation explained in Development page).

  2. Registration Page

    • The registration page is in path “www/register.html”. It contains both the HTML5 and Javascript code used.
    • The registration function contains an HTML5 form with a request to the Drupal RESTful API. (Implementation explained in Development Page).
    • Can be easily modified to add as many registration fields as necessary.

  3. Forgot Password Page

    • The registration page is in path “www/forgot.html”. It contains both the HTML5 and Javascript code used.
    • Requires the user to input an email so that the system can send a new password request there.
    • The forgot password function contains an HTML5 form with a request to the Drupal RESTful API. (Implementation explained in Development Page).

  4. Main Page

    • Contained in path “www/index.html”. It only contains the HTML5.
    • The Javascript and jQuery functions are in in path “www/js/functions.js”. They are mainly AJAX requests to the Drupal RESTful API. They inject HTML5 classes in “index.html” after they are executed.
    • Contains multiple divs that are hidden and shown when the menu buttons are pressed (via jQuery).
    1. Home page

      • The Home page contains an AJAX request to get Headlines, Latest News, Recently Added. They all dynamically add HTML5 code to index.html.
      • There is also a refresh button that recalls all these requests.

    2. Search page

      • The search page contains a form that contains the search key, category and filters. This also submits an AJAX request to the Drupal Restful API and uses jQuery to show the results.
      • There is also a form for the user to contribute resources.
      • The Quick Search function makes a call to the REST API and returns the user’s preferences. Then it sends an AJAX request to get the necessary data.

    3. In Focus page

      • Some more data will be added later from an older iNRIC resource.
      • AJAX Request that obtains Training links.

    4. Links page

      • Static links for Conferences
      • AJAX Request that obtains Useful Links from REST API.

    5. Profile page

      • AJAX Request that saves User Filters to database. (uses session token)
      • AJAX Request that changes email or country. (uses session token)
      • AJAX Request that changes password. (uses session token)
      • Log out button. (Logout function uses session token).

  5. Notifications

    • Notifications are implemented (for Android) using PUSHWOOSH in path “www/js/notifications.js”. They are currently not being used as Phonegap has issues with loading the plugin in the app.

  6. Instructions to run are on Development page.
