Research

Platforms and API's, App Design, Database Design

Platforms and API's

Initial research regarded iOS and Android App Development as well as Drupal Database integration. However the integration of Drupal in native apps proved to be difficult and the development on two different platforms seemed unrealistic for two people with no previous experience in either iOS or Android development.

Due to the need of a multi-platform solution and the usage of a Drupal database (main website), the application is built with Adobe PhoneGap (Powered by Cordova). This application transforms web-apps into applications.

Front end

  • HTML5 - Markup language also used for mobile websites
  • CSS3 - Cascading Style Sheet language with transitions, gradients and animations to avoid usage of Flash
  • Javascript (jQuery) - High level, interpreted programming language used to program web pages (with jQuery framework)
  • PushPlugin for PhoneGap to handle notifications

Furthermore, we researched and also integrated the use of jQuery mobile. However, this proved to have clash issues with our custom CSS and would also create transition issues when keeping the header and navigation menu fixed (to give the impression of a native app). Additionally, the use of jQuery mobile has the drawback of loading each page separately without tweaking, whereas the use of HTML5 divs and hiding them with the use of CSS3 would increase the loading time when initially opening the apps, but would make transitions between pages instantaneous.

Main Design

The main idea of the project is for the user to be able to use both the website and the app and have his Preferences saved in the Database. As iNRIC is mainly a collection of links to articles and news sites, most of the reading will happen in a separate browser window. The main need is for the app to filter Articles and Searching with the user Preferences.
