Testing

Automated and user-based

Automated testing

The application is coded as a Web-App in HTML5, CSS3 and Javascript and has the addition of the DrupalGap module for database integration. Automated testing can be done for the web-app itself before being ported with Phonegap.

This can be done with multiple web application automation tools to check for security, accesibility, functionality and compatibility issues. Furthermore, they can be used on the main website to check for any errors when implementing the user profiling.

  • Loadster - Loading Stress Test
  • Sahi - Functional/Regression Testing
    • -- No longer used. Issues with installation.
  • Link Tiger - Broken Link testing
    • -- No longer used as links are managed by Content Manager.
  • As the iNRIC website was built by another developer and the database was tested, we decided not to run Drupal tests on the Database as we made no modifications to it.

In order to test the iNRIC app after it has been built with Phonegap, we can use Testmunk. It provides the ability to write test cases on real devices across multiple platforms. Furthermore, it reviews UI design and provides numerous already created tests that check reliability, speed, erorrs etc. (Recommendation)

Loadster Workbench results

In order to test the loading times of the app, we used Loadster with 25 virtual users to simulate the AJAX requests of the app.

You can find the full report here.

These results show that the average loading time of the app login plus opening all tabs (majority of ajax requests) is less than 2 seconds per request. This shows that this solution is viable and fast. Furthermore it shows that loading everything at once would have significantly increased the loading times.

User Testing

Is perhaps even more important than automated testing, as the app is meant for a large number of technical and non-techinal individuals. Therefore, the application should be easy to use and should work across a large number of devices.

A/B Testing

We conducted various small scale A/B Tests with around 10 users using various screen sizes and platforms.

Tests conducted were on:

  • 2 x iPhone 6S Plus
  • iPhone 6 Plus
  • iPhone 6
  • 2 x iPhone 5S
  • iPhone 4S
  • Samsung Galaxy Note 3
  • Nexus 5
  • Samasung Galaxy S6

The first purpose was to test the front-end and how the user would interact with various elements within the app (while having previously used the main iNRIC website).

The second purpose was to test functionality such as searching and loading times. (All data loaded at once, versus data loaded on demand).

The final purpose was to also see how the app would respond and function on different screen sizes and platforms.

The test subjects:

  • 2 x Medicine Students (aged 20-21)
  • 3 x Computer Science Students (aged 20-21)
  • 2 x Doctors (aged 55- 65)
  • 1 x Nurse (aged 57)
  • iPhone 4S
  • 2 x Engineers (aged 45-50)

Test subjects were chosen from different age groups as to see make the app accessible to as many people as possible. Furthermore, more people that are studying/have studied Medicine have been chosen as the iNRIC website as well as the app is directed more towards medical staff.

Test subjects were split into two groups.

Group 1
  • 1 x Medicine Student
  • 2 x Computer Science Students
  • 1 x Doctor
  • 1 x Engineer
Group 2
  • 1 x Medicine Student
  • 1 x Computer Science Students
  • 1 x Doctor
  • 1 x Engineer
  • 1 x Nurse
Test No. 1

The first test conducted regarded the front-end of the app as it supposed to mimic the color scheme and some aspects of design from the iNRIC website. We therefore set up one version of the app with a simple white background and light orange for links and another with a gradient grey to light grey background and dark orange for links. We gave the first version of the app (with the white background) to group 1. We gave the second version to group 2 and we asked them to read through the headlines. People read less on the first version of the app and did not realise that the titles are clickable, whereas on the second version, they read more with some subjects even clicking the links.

Test No. 2

The second test regarded font-size. The first group was given a font-size that mimics the one on the iNRIC website whilst the second group was given a a slightly larger font. Both font-sizes adjusted to various screen sizes. They were again asked to read through the headlines. Most of the people from the first group read significantly less with some(aged 55-65) even trying to zoom in on the text, than the ones from the second group.

Test No. 3

The last test regarded the loading times of the app. The data from the application is being requested from the iNRIC Rest Service. The first group was asked to log in to the app that was loading all the website data in one go but provided quicker access after the initial load time, whereas the other group was given the version of the app that was loading data when they would switch tabs within the app. These tests revealed that subjects in the first group did not have the necessary patience to wait for the app to load, with some even trying to restart it.

Development Testing

Initial testing will be done by the developers across multiple devices to check for screen resizing and other compatibility issues.

We used Safari Responsive Design mode, Google Chrome Responsive Design mode across multiple screen resolutions.(Regular resolution or HiDPI resolution) to test the front-end and back-end. We also used POSTMAN as well as simply cross-check with the iNRIC website for the correct data to be sent via AJAX requests.

Local device testing

The application was also built with Phonegap CLI locally and sent over to Android and iOS devices with the use of the Phonegap Developer app. Therefore, the app was tested by the team and our client on

  • iPhone 6S Plus
  • iPhone 6 Plus
  • Galaxy Note 3
  • Nexus 5
  • iPhone 6

App Store and Google Play deployment Testing

After the app has been deployed, testing is done through usage, as a portion of users normally report uncaught bugs or compatibility issues.
